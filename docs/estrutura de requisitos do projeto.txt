estrutura de requisitos do projeto
introdução do projeto:
O projeto consiste no desenvolvimento de uma aplicação web para gestão de inventário informático. A ideia é criar um sistema onde seja possível registar e organizar computadores de forma simples, agrupando-os em diferentes inventários, que podem representar uma rede, uma sub-rede ou apenas um conjunto lógico de dispositivos. A aplicação vai permitir adicionar equipamentos manualmente, consultar informação relevante como IP, hostname ou estado, e gerir esses dados através de uma interface acessível por browser.

Para além da gestão manual, o sistema também inclui funcionalidades ligadas à área de redes, como a realização de scans para identificar dispositivos ativos dentro de um intervalo de IPs, permitindo depois adicionar esses dispositivos ao inventário. O projeto também prevê autenticação de utilizadores, controlo de acessos e registo de logs, incluindo logs de segurança e RDP. No geral, o objetivo é criar um sistema simples mas funcional, que combine conceitos de desenvolvimento web com algumas funcionalidades práticas relacionadas com redes.

Objetivo mínimo (MVP):
O objetivo mínimo do projeto é desenvolver uma versão funcional do sistema que permita a autenticação de utilizadores, a criação e consulta de inventários, o registo manual de computadores e a consulta básica do inventário através de uma interface web. Para além disso, o sistema deverá incluir uma funcionalidade inicial de scan de rede, capaz de identificar dispositivos ativos num intervalo de endereços IP e apresentar os resultados ao utilizador autorizado.

Requisitos 
O sistema deverá permitir a gestão de inventário informático de forma centralizada, segura e organizada. A aplicação deverá ser disponibilizada através de um web server, comunicar com um web service responsável pela lógica da aplicação e armazenar a informação numa base de dados. O sistema deverá ainda incluir mecanismos de autenticação, autorização e registo de logs, sendo executado em ambiente Docker. Além disso, o sistema deverá permitir a consulta de logs de segurança e logs RDP provenientes dos computadores monitorizados, a descoberta automática de dispositivos na rede através de mecanismos de scan e a disponibilização de uma interface frontend básica para interação com o utilizador. O sistema deverá também permitir a organização de equipamentos em diferentes inventários, associados a sub-redes ou grupos lógicos distintos

Regras de negócio

Cada computador deve possuir um identificador único.
Cada computador deve possuir um número de série único.
Cada computador deve ter sempre um estado definido.
Apenas utilizadores autenticados podem aceder ao sistema.
Apenas utilizadores autorizados podem criar, editar ou remover computadores.
Apenas administradores podem gerir utilizadores.
Apenas utilizadores autorizados podem executar a funcionalidade de scan de rede.
Todas as ações críticas devem ser registadas em log.
Cada computador pode estar associado a uma localização.
Cada computador pode estar associado a um utilizador responsável.
A remoção de um registo deve ficar refletida nos logs do sistema.
O sistema não deve permitir a criação de registos incompletos nos campos obrigatórios.
Cada log de dispositivo deve estar obrigatoriamente associado a um computador.
O sistema deve permitir a consulta de logs de dispositivo apenas a utilizadores autorizados.
Os logs recolhidos dos computadores devem conservar, no mínimo, a data, o tipo de log e a descrição do evento.
Os logs de dispositivo considerados pelo sistema devem restringir-se a logs de segurança e logs RDP.
Cada utilizador deve poder consultar apenas o seu próprio histórico de ações, salvo utilizadores com perfil de administrador.
Os administradores devem poder consultar os logs de todos os utilizadores do sistema.
Os dispositivos descobertos automaticamente devem poder ser adicionados ao inventário.
Cada dispositivo descoberto deve ser identificado, no mínimo, pelo seu endereço IP.
Os dispositivos devem ter indicação da origem do registo, manual ou automática.
Cada computador deve estar associado a um inventário.
Cada inventário pode agrupar computadores de uma sub-rede ou grupo lógico específico.
Os dispositivos descobertos através de scan podem ser associados a um inventário.
Apenas administradores podem criar, editar e remover inventários.
O scan não deve criar computadores manualmente registados de forma automática.
A identificação de duplicados em dispositivos descobertos deve considerar, no mínimo, inventário + IP.
Quando um dispositivo não for encontrado em scans subsequentes, deve poder ser marcado como inativo mantendo histórico da última vez ativo.
